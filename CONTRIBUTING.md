# Contributing to stockclaw

## 中文

感谢你对 `stockclaw` 的贡献。

### 提交流程

1. Fork 仓库并创建功能分支。
2. 完成代码修改并保证本地可运行。
3. 提交 Pull Request，清晰描述变更内容和原因。

### 建议规范

- 保持改动聚焦，避免在同一个 PR 中混入无关修改。
- 新增功能时，优先补充必要文档。
- 涉及 API 或行为变更时，说明兼容性影响。

## English

Thanks for contributing to `stockclaw`.

### Workflow

1. Fork the repository and create a feature branch.
2. Implement your changes and verify locally.
3. Open a Pull Request with a clear description of what changed and why.

### Guidelines

- Keep changes focused and avoid unrelated edits in the same PR.
- Add or update documentation for new features.
- Document compatibility impact for API or behavior changes.
